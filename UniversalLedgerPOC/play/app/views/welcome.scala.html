What Can You Do?

There are currently planned four parts to this system:

  1. Mockups of Initial Set Up Screens
  2. Mockups of Initial Stages in Transaction Processing
  3. File-based Conversion and System Initiation Processes
  4. Integrated Posting, New Event Generation, and Reporting Processes
Each of these are described below. Links are shown at right.

Initial Setup Mockups

These initial screens, Universal Ledger Screen Mockups, show what initial user ID creation and ledger setup would mean. The end results of these screen would be a set of meta data defining the universal ledger to those participating parties. This data would be used to generate Scala code which could be deployed to servers to create and run a Universal Ledger

Initial Transaction Processing

These screens, Processing Screen Mockups, show how processing begins after the Universal Ledger has been created. They include selecting files for processing or importing transactions, and then the results of identifying new Involved Parties in the processed transactions.

Start the POC

Limited functionality has been created to demonstrate the capabilities of a Universal Ledger. You can test them out beginning here: File-based Conversion and System Initiation Processes
